Explicación

La práctica ha consistido en una tienda online responsive en código html y css. La página debía contener un menú de navegación, un slider, algunos enlaces, algunos artículos e información diversa de la tienda.

En lo referente a los estilos, he empezado con un código extraído en la web github (reset.css) que resetea todos los estilos propios de un navegador. De este modo, maquetaba con mis propios estilos (styles.css) el diseño exacto para cualquier navegador. En styles.css he importado la fuente y las demás hojas de estilo.

El menú de navegación consta de un logo que escala la imagen ante la acción hover y un menú navbar clásico con subrayado dinámico ante la acción hover. He utilizado un breakpoint en un ancho de 600px ya que el navbar se desalineaba.

El slider no tiene ningún secreto, he utilizado para la imagen un width de 100% (ventana del navegador) y para la descripción un max-width de 1200px centrado.

El main se divide en rows con sus respectivas columns. He diseñado un estilo similar a bootstrap para maquetar el porcentaje de las cajas según el ancho. Creo que si no se utiliza javascript deben utilizarse mediaqueries para cada ancho. Además, he utilizado la propiedad display flex para que todas las columnas tuviesen la misma altura por row y algunas imágenes fluyesen. El único problema es que con display flex las columnas no se rompen y no se realizan los breakpoints, por eso he utilizado la flex-basis que permite ejecutar los floats para cada breakpoint.

Por último, comentar que he utilizado las propiedades clear, content y display en el after de los divs en lugar de overflow. ¿Por qué? Es verdad que en esta práctica no hay ninguna diferencia, pero si tuviéramos algún div tipo menú/submenú que sobrepasara la altura del contenedor y apareciera mediante javascript, el overflow hidden lo cortaría.
